Recursive Governance

This page implements recursive human governance over autonomous agents. You set the rules, agents follow them, the auditor checks compliance, and the system improves over time -- always with you in control.

The core idea: agents do work, but they also produce feedback about what went wrong and what could be better. That feedback becomes policy proposals that you review and approve. Each cycle tightens the rules and improves agent performance.

Board Directives

Rules you issue as a board member. The auditor reads these and reconciles them with actual agent behavior. Toggle on/off as needed. These are your constitutional layer.

Run Audit

Triggers the auditor agent to review completed work -- retry counts, rejection rates, cost patterns, archetype gaps. It produces suggested policies and proposes archetype changes.

Suggested Policies

Agent-suggested rules from audit findings. You accept or reject each one. Accepted policies are loaded by all agents automatically. Short, actionable, 1-2 sentences.

Archetype Updates

Proposed changes to agent role definitions. You review a diff and approve or reject. Approved changes shape all future agent runs.

The loop: you set directives → agents work → audit reviews patterns → suggests improvements → you approve or reject → agents perform better → next audit finds new patterns. Each cycle compresses organizational learning into actionable rules, with human approval at every step.
